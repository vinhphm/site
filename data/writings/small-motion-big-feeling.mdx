Good UX doesn't need to be dramatic. The Google UX Design course puts it well:

> "If a product is enjoyable, it means the design delights the user. The design reflects what the user may be thinking or feeling and creates a positive connection with them. A product's design doesn't have to be enjoyable for it to function properly. But, an enjoyable design adds to an already functional product and can enhance the user's feelings about the experience."

**Micro-interactions** are one of the most effective ways to get there: small, purposeful animations tied to a single action. The kind of thing users don't consciously notice but would immediately miss if it were gone.

Here are two I added to my site, plus one media format change that quietly improved everything else.

## Ditch the GIFs

While working on these animations I ran a Lighthouse audit and hit a flag I'd been ignoring: GIFs. I'd been using screen recordings saved as GIFs, and some had ballooned to several MB, slow to load, with that blank-screen delay while they buffered.

The answer is MP4 or WebM. Lighthouse even has an article on it: [Use video formats for animated content](https://developer.chrome.com/docs/lighthouse/performance/efficient-animated-content/). Dramatically smaller files, better compression, no blank screen.

The basic switch is simple:

```html
<video autoplay loop muted playsinline src="my-animation.mp4"></video>
```

But in Astro (and other SSGs), a plain `<video>` tag causes problems in Firefox, requiring a page reload to play. The fix is a wrapper component that resolves the URL at runtime:

```vue title="src › components › Video.vue"
<script setup lang="ts">
import { onMounted, ref } from 'vue'

const props = defineProps<{
  src: string
}>()

const videoSrc = ref<string | undefined>()

function getVideoUrl(videoPath: string): string {
  return new URL(videoPath, import.meta.url).href
}

onMounted(() => {
  videoSrc.value = getVideoUrl(props.src)
})
</script>

<template>
  <video v-if="videoSrc" :src="videoSrc" autoplay loop muted playsinline />
</template>
```

Use it in MDX like this:

```mdx
import themeTransitionAnimation from '@/assets/videos/2024/theme-transition-animation.mp4'
import Video from '@/components/ui/Video.astro'

<Video src={themeTransitionAnimation} />
```

The difference from a static `import` is that `import.meta.url` constructs the URL dynamically at runtime, which is what makes Firefox happy.

## It all adds up

None of these changes is dramatic on its own. A smooth hamburger animation, a satisfying theme toggle, videos that load instantly, each one is a small detail. Together they're the difference between a site that feels finished and one that feels assembled.
